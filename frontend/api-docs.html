MTA Arrivals API

Swagger-style reference for device + web clients.
GET /device/setup
First-time setup: provide zip code or station name to get choices.
Example requests
GET /device/setup?zip=11229
GET /device/setup?station=brighton
Example response (zip)
{
  "mode": "zip",
  "zip": "11229",
  "centroid": { "lat": 40.6012, "lon": -73.9448 },
  "stations": [
    {
      "id": "B11",
      "name": "Brighton Beach",
      "stopIds": ["B11N", "B11S"],
      "routes": ["B", "Q"],
      "lat": 40.5776,
      "lon": -73.9615,
      "distance_miles": 1.23
    }
  ]
}

GET /stations
All stations (name, ids, routes, lat/lon).
Example request
GET /stations
GET /stations/search
Search stations by partial name.
Example request
GET /stations/search?query=brighton
GET /stations/by-id
Get station by id.
Example request
GET /stations/by-id?stationId=B11
GET /zipcode-info
ZIP centroid + nearest stations.
Example request
GET /zipcode-info?zip=11229
GET /arrivals
Arrivals for one stop id and selected routes.
Example request
GET /arrivals?stopId=B11N&routes=Q,B
GET /alerts
Alerts filtered by routes/stops/stations.
Example request
GET /alerts?routes=Q,B&stopIds=B11N,B11S&stationIds=B11
GET /weather
Weather payload for a lat/lon.
Example request
GET /weather?lat=40.6012&lon=-73.9448
GET /reverse-geocode
Reverse geocode (borough, city, etc.).
Example request
GET /reverse-geocode?lat=40.6012&lon=-73.9448
